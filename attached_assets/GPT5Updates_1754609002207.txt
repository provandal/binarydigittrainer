1) Decide: softmax + cross-entropy or sigmoid + binary cross-entropy (BCE) — and make them consistent
Right now your code uses two independent sigmoids on the outputs, but the “cross-entropy” loss you compute is the softmax style -Σ t_k log(a_k) (only the positive term). Then in backprop you use the gradient simplification δ = a − t (which is correct for softmax+CE or sigmoid+BCE), but the loss math and the activation don’t match. That combination makes one bias run away trying to satisfy a loss it can’t actually minimize properly.

Pick one path and update both the forward and the loss/grad:

Option A (recommended): softmax + cross-entropy (multi-class, mutually exclusive)
Forward output: compute softmax of the two logits.

Loss: CE = -Σ t_k log(p_k).

Output delta: δ_k = p_k − t_k (no sigmoid derivative factor).

// utils
const softmax = (z: number[]) => {
  const m = Math.max(...z);
  const exps = z.map(v => Math.exp(v - m));
  const sum = exps.reduce((a,b)=>a+b, 0);
  return exps.map(e => e / sum);
};

Option B: two sigmoids + BCE (one-vs-rest)
Keep your sigmoids.

Loss: BCE = -Σ[ t_k log(a_k) + (1−t_k) log(1−a_k) ].

Output delta: δ_k = a_k − t_k (still no extra σ’ factor).

2) Don’t multiply by σ′ at the output when using CE/BCE
You already do δ = a − t in the CE branch — great. Make sure you do not multiply that by sigmoidDerivative(z) at the output; that double-counts and will blow up updates.

For the hidden layer, keep using:

objectivec
Copy
Edit
δ_hidden = (W_out^T · δ_out) ⊙ σ′(z_hidden)
But compute δ_out first. (You can snapshot W_out before you update it if you want strict “standard” backprop, but using the just-updated matrix usually won’t cause the instant divergence you’re seeing.)
